- 2026-06-05: В agent.py нужно поправить поведение агента при ошибках. Проблема: когда tool возвращает ошибку (например «API недоступен» или ошибка 404), агент всё равно пишет в Action, что действие выполнено успешно (например «Создал пользователя»), хотя на самом деле ничего не произошло. Это противоречит полю Status: error.

  Исправь системный промпт так, чтобы агент соблюдал правила согласованности ответа:

  Если tool вернул ошибку или сообщение о недоступности API — агент НЕ должен утверждать, что действие выполнено. Status должен быть error, а Action должен описывать ПОПЫТКУ, а не успех. Например: «Попытка создать пользователя не удалась», а НЕ «Создал пользователя».

  Status, Action, Data и Errors должны быть согласованы между собой:

  Если Status: error — поле Data должно быть прочерком (—), а Errors заполнено.
  Если Status: success — Errors должно быть прочерком (—), а Data заполнено.
  Никогда не пиши в Action об успешном действии, если tool вернул ошибку.
  Агент НИКОГДА не выдумывает результат. Если tool не вернул реальных данных (например пользователь не создан из-за ошибки) — агент не должен сочинять id или подтверждать создание. Он опирается ТОЛЬКО на то, что реально вернул tool.

  Добавь в системный промпт few-shot пример ошибочного случая, чтобы модель не сбивалась. Например:
  Запрос: «создай пользователя Anna», tool вернул «API недоступен».
  Правильный ответ:
  Status: error
  Action: Попытка создать пользователя Anna не удалась
  Data: —
  Errors: API недоступен — не удалось подключиться к серверу.

  После правки проверь два случая и покажи мне оба:

  успешный (при запущенном API) — создание пользователя;
  ошибочный (например с несуществующим пользователем get_user 9999) — агент должен честно сообщить об ошибке, не выдумывая успех.
